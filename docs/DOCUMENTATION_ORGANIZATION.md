# Documentation Organization

This document describes the organization structure of all markdown documentation files in the repository.

## Root Directory

The following files remain in the root directory as they serve as primary project documentation:

- `README.md` - Main project README with setup and usage instructions
- `AGENTS.md` - AI agent configuration and operational instructions
- `CLAUDE.md` - Claude AI configuration and guidelines

## Documentation Structure

All other documentation has been organized into the following folders under `docs/`:

### 📊 `docs/reports/`
Status reports, completion summaries, and various project reports documenting the development and deployment process.

**Contains:**
- Status reports (PROJECT_STATUS.md, PRODUCTION_STATUS_SUMMARY.md, etc.)
- Completion summaries (COMPLETION_REPORT.md, SESSION_COMPLETION_SUMMARY.md, etc.)
- Quality reports (CODE_QUALITY_IMPROVEMENTS.md, LINTING_FIXES_SUMMARY.md, etc.)
- Database reports (DATABASE_SCHEMA_UPDATE.md, SCHEMA_ISOLATION_SUMMARY.md, etc.)
- Production reports (PRODUCTION_READINESS_ASSESSMENT.md, PRODUCTION_READY_SUMMARY.md, etc.)

### 🚀 `docs/deployment/`
All deployment-related documentation, guides, and status reports.

**Contains:**
- Deployment guides (DEPLOYMENT.md, PRODUCTION_DEPLOYMENT.md)
- Deployment checklists (DEPLOYMENT_CHECKLIST.md)
- Production documentation (README_PRODUCTION.md)
- Deployment reports (DEPLOYMENT_COMPLETE_SUMMARY.md, FINAL_DEPLOYMENT_REPORT.md, etc.)

### 🧪 `docs/testing/`
Testing guides, test reports, and verification documentation.

**Contains:**
- Testing guides (BROWSER_TESTING_GUIDE.md, QUICK_TESTING_GUIDE.md)
- Test reports (COMPLETE_TESTING_REPORT.md, SMOKE_TEST_REPORT.md, FUNCTIONAL_TESTING_REPORT.md)
- Verification reports (FEATURE_VERIFICATION_REPORT.md, LINK_VERIFICATION_REPORT.md, PRE_LAUNCH_QA_REPORT.md)
- Training testing (TRAINING_CONTENT_INSPECTION.md, TRAINING_ROUTES_VERIFICATION.md)

### 🔧 `docs/implementation/`
Implementation summaries and guides for specific features and integrations.

**Contains:**
- Feature implementation summaries (FREE_TRIAL_IMPLEMENTATION_SUMMARY.md)
- Integration guides (STRIPE_CHECKOUT_IMPLEMENTATION_SUMMARY.md)
- Policy setup (STRIPE_CHECKOUT_POLICIES_SETUP.md)

### ✅ `docs/compliance/`
Compliance reports, policy verification summaries, and security documentation.

**Contains:**
- Compliance reports (FTC_COMPLIANCE_REVIEW_SUMMARY.md)
- Policy verification (E_COMMERCE_POLICY_VERIFICATION_SUMMARY.md, E_COMMERCE_POLICY_ALIGNMENT_REPORT.md)
- Security documentation (RANSOMWARE_THREAT_DASHBOARD.md)

### 🌐 `docs/translation/`
Translation implementation summaries, verification reports, and translation reviews.

**Contains:**
- Translation implementation (TRANSLATION_IMPLEMENTATION_SUMMARY.md)
- Translation verification (TRANSLATION_VERIFICATION_SUMMARY.md)
- Translation reviews (FRENCH_TRANSLATION_REVIEW.md)
- Untranslated content reports (UNTRANSLATED_TEXT_REPORT.md)

### ⚙️ `docs/configuration/`
Configuration guides and setup instructions for various services and tools.

**Contains:**
- Service configuration (SUPABASE_CONFIG.md)
- Multi-project setup (MULTI_PROJECT_SUPABASE_SETUP.md)
- Monitoring setup (MONITORING_SETUP_GUIDE.md)

### 📁 Other Documentation

Additional documentation files in `docs/` root:
- `DEPENDENCY_MAPPER_ENHANCEMENTS.md` - Dependency mapper feature documentation
- `FREE_TRIAL_SETUP_GUIDE.md` - Free trial setup guide
- `I18N_IMPLEMENTATION_PLAN.md` - Internationalization implementation plan
- `INTERNAL_LINKING_ANALYSIS.md` - Internal linking analysis
- `INTERNAL_LINKING_STRATEGY.md` - Internal linking strategy
- `MISSING_CONTENT_REPORT.md` - Missing content report

## File Naming Conventions

- `*_SUMMARY.md` - Summary reports of completed work
- `*_STATUS.md` - Current status reports
- `*_REPORT.md` - Detailed inspection and assessment reports
- `*_FIXES.md` - Reports documenting fixes and improvements
- `*_COMPLETION.md` - Completion checklists and summaries
- `*_GUIDE.md` - Step-by-step guides and instructions
- `*_VERIFICATION.md` - Verification and validation reports

## Notes

- All documentation has been preserved - no information was deleted during reorganization
- Each subfolder contains a README.md file describing its contents
- Policy files remain in `public/policies/` as they are part of the public website
- Download templates remain in `public/downloads/` as they are user-facing resources
